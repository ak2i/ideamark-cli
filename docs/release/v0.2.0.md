# ideamark-cli v0.2.0

## Highlights

- Rebased the CLI onto IdeaMark Core `v1.1.1`.
- Dropped the previous `v1.0.3` compatibility-first direction for the v0.2.0 track.
- Added new `tests/fixtures/v1.1.1/` baseline samples.
- Updated `validate` to enforce v1.1.1 structural constraints:
  - required `entities`, `occurrences`, `sections`
  - required `entity.payload`
  - `payload.body | payload.ref | payload.cache` one-of requirement
  - `payload.ref.uri` requirement
  - `occurrence.entity` / `occurrence.role` requirement
  - `section.occurrences` non-empty requirement
  - `relations.from/to` structural reference validation
- Updated `extract`, `compose`, `render`, `diff`, `describe`, and `ls` to use v1.1.1-oriented registry shape.
- Added `npm run smoke:v0.2.0`.

## Breaking Changes

- The v0.2.0 track treats IdeaMark Core `v1.1.1` as the primary document model.
- Old `anchorage`-centric validation assumptions are no longer required by `validate`.
- Entity payload now follows the v1.1.1 `payload` structure instead of older `content`-centric forms.
- `relations` and `perspectives` are treated as mapping-style registry namespaces in current tooling.

## Test Runners

- Smoke:
  - `npm run smoke:v0.2.0`
- Historical runners retained:
  - `npm run smoke:v0.1.3`
  - `npm run llm-metrics:v0.1.3`

## Notes

- The v0.2.0 smoke runner currently calls core modules directly because the current sandbox environment is restrictive for nested Node process execution.
- Some internal `node:test` suites still reflect older helper assumptions and should be refreshed incrementally on top of the new fixture baseline.
