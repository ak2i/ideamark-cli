# Release v0.1.2

## Summary
- Aligned `describe` output with doc-cli-contract v1.0.2 (default `md`, updated capabilities JSON/MD).
- Added `--version --format json` for machine-readable version metadata.
- Enabled `describe ai-authoring --format json` (template-based output).
- Refreshed v1.0.2 dev/spec docs for Evidence/Anchorage and updated template tool versions.

## Changed
- `describe` default `--format` is now `md` (including `describe params`).
- `describe capabilities --format json` now matches the v1.0.2 schema (contract/tool metadata, commands/options).
- `describe capabilities --format md` now follows the v1.0.2 template.
- `describe ai-authoring --format json` returns JSON from `ai-authoring.json`.

## Added
- `ideamark --version --format json` (includes `tool.version`, `contract.version`, `document_spec.version`).

## Docs
- Updated v1.0.2 Evidence/Anchorage dev/spec documents for consistency.
- `params.json` / `ai-authoring.json` tool versions updated to `0.1.2`.

## Notes
- v0.1.1 behavior is preserved aside from the `describe` default format change and new JSON outputs.
