# ideamark-cli v0.1.3 (development)

## Highlights

- Added orthogonal `describe` interface options:
  - `--audience human|ai`
  - `--lang ja|en|ja-JP|en-US`
  - `--model small|large`
  - `--profile <alias>`
- Added `describe` discovery topics:
  - `describe ls --target guides`
  - `describe routing`
- Added `lint` command (diagnostic-first, non-blocking default).
- Added `diff` command (YAML-first structural diff).
- Updated `describe capabilities` for Doc CLI Contract v1.0.3 alignment.
- Added v0.1.3 smoke runner and LLM metrics runner.

## New commands

### lint

```bash
ideamark lint [<infile>|-] [--format ndjson|json|md] [--strict] [--profile minimal|diagnostic|strict]
```

Behavior:
- default: non-blocking (`exit 0` even with error diagnostics)
- `--strict`: fail on error diagnostics (`exit 1`)

### diff

```bash
ideamark diff <from> <to> [--format ndjson|json|md] [--scope yaml|all] [--include-markdown] [--include-meta]
```

Behavior:
- default: YAML-first structural diff
- excludes timestamp/meta fields by default

## Test runners

- Smoke:
  - `npm run smoke:v0.1.3`
- LLM metrics:
  - `npm run llm-metrics:v0.1.3`
  - optional artifact: `node tests/run-llm-metrics-v0.1.3.js --out <path>`

## Notes

- `compose` remains the integration command; full merge policy implementation is still a non-goal for v0.1.3.
- Breakdown round-trip testing is provided as test scaffolding, not a finalized merge feature.
