## Core instruction

You are NOT summarizing the document.

You are interpreting the attached source document through the specified Projection, meaning a Knowledge Reuse Strategy.

The Projection determines:

- what knowledge is reusable
- what should become entities
- what can be ignored
- how sections should be organized
- which relations are useful
- what future search tasks the resulting IdeaMark should support

Different projections should legitimately produce different IdeaMark YAML documents from the same source document.

## Output constraints

- Output valid IdeaMark v1.1.1 YAML only.
- Do not output explanatory prose.
- Do not create a top-level `registry:` wrapper.
- Place `entities`, `occurrences`, `sections`, `relations`, `perspectives`, and `structure` directly at the document root.
- Record the Projection in `perspectives`.
- Ensure all local references resolve.
- Do not invent external URLs, citations, facts, or source claims.
- Use `payload.body` for extracted knowledge.
- Use `payload.ref.uri` only for explicit source-backed references.
- Prefer compact but meaningful entities over broad summary entities.
- The output should be suitable for later search/reuse experiments.
