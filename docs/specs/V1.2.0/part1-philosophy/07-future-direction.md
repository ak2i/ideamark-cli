# 7. Future Direction

**Part:** 1 — Philosophy  
**Status:** Draft  
**Type:** Informative / Future Work

Projection specifications, coordinate models, retrieval architectures, and authoring methodologies evolve independently while remaining interoperable with IdeaMark Core.

## Projection

Projection is expected to become an independent reusable intellectual asset.

Projection defines how original sources are converted into reusable intellectual activity structures and how future retrieval should be supported.

## Coordinate Models

Coordinate models describe how Sections are positioned within intellectual activities.

Core should define how such coordinate claims are represented, but should not define a universal coordinate vocabulary.

## Retrieval Architecture

IdeaMark retrieval is expected to compare situations, projections, sections, occurrences, and original-source references rather than rely only on keywords or raw document similarity.

## Authoring Methodology

IdeaMark authoring is expected to involve humans, AI, validation tools, and iterative refinement.

Projection authoring and IdeaMark authoring should therefore be treated as coupled but separately evolving practices.
