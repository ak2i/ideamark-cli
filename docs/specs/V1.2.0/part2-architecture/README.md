# Part 2 — Architecture of Human-AI Co-evolution

**Version:** IdeaMark Core v1.2.0  
**Status:** Planned

This part describes the architecture through which humans, AI systems, projections, IdeaMark documents, retrieval systems, and authoritative original sources participate in continuous co-evolution.

In this specification, co-evolution means the continuous mutual development of humans and AI through shared intellectual activities grounded in authoritative original sources.

Part 1 explains why IdeaMark uses engineering through separation: reusable structure is separated from meaning so that it can be managed and reused without fixing interpretation.

Part 2 explains engineering through reconstruction: how Original Source, Projection, IdeaMark, retrieval, AI interpretation, human interpretation, and human action together reconstruct meaning and produce new intellectual activities.

## Key Architectural Flow

The basic reconstruction flow is:

```text
Original Source
    ↓
Projection
    ↓
IdeaMark
    ↓
Retrieval / Navigation
    ↓
AI Interpretation
    ↓
Human Interpretation
    ↓
Human Action
    ↓
New Original Source
```

This flow is not intended to be a fixed workflow.

It is an architectural cycle that explains how meaning may be reconstructed and how future intellectual activities may become new sources for reuse.

## Scope

Part 2 should explain the system-level reconstruction flow before the Core Model is specified.

It should not define YAML fields, validation rules, or implementation-specific retrieval algorithms.
